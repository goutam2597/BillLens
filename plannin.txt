# Smart Receipt Scanner AI App

# Complete Development Plan Using Flutter BLoC Architecture

Version: 1.0

---

# Main Goal

Build a production-ready **offline-first AI receipt scanner mobile app** using **Flutter + BLoC Architecture**.

The app will help freelancers and small business owners scan receipts, extract expense data, categorize expenses using AI, save data offline, and sync with a PHP/MySQL backend when internet is available.

---

# Core Features

* User authentication
* Offline-first expense management
* Receipt scanning
* Receipt image cropping
* OCR text extraction
* AI expense categorization
* Manual expense entry
* Expense list and details
* Category management
* Analytics and reports
* PDF/CSV export
* Cloud sync
* Subscription system
* Google AdMob ads for free users
* No ads for premium users

# App Development Phases

## Phase 1: Flutter UI Design

Build all screens with dummy/local data first.

## Phase 2: Flutter BLoC Architecture Setup

Create BLoC, Events, States, Repositories, Models, and Use Cases.

## Phase 3: Offline Database Integration

Save all important data locally first.

## Phase 4: Receipt Scanner + OCR + AI Flow

Scan receipt, extract data, categorize expense, and save locally.

## Phase 5: PHP/MySQL Backend

Create REST API, authentication, sync system, upload system, and AI processing.

## Phase 6: Sync System

Sync local data with backend when internet is available.

## Phase 7: Ads + Subscription

Show ads for free users and remove ads for premium users.

## Phase 8: Testing + Release

Test offline, online, sync, ads, authentication, reports, and release.

# Offline-First App Flow

The app must always save data locally first.

Flow:

User scans receipt
↓
Receipt image saved locally
↓
Extracted data saved locally
↓
Expense appears immediately in app
↓
If internet is available, sync with server
↓
If internet is unavailable, add item to sync queue
↓
Sync automatically later

# Backend Security Requirements

* Use password_hash for password storage
* Use password_verify for login
* Use JWT token for protected APIs
* Validate all inputs
* Sanitize file uploads
* Restrict upload file types
* Limit receipt image size
* Enable CORS properly
* Return JSON response for all APIs
* Never expose PHP errors in production

---

# Offline Sync Rules

## Create Expense Offline

* Save expense in SQLite
* Set sync_status = pending
* Add item to sync_queue
* Show expense immediately
* Upload when internet is available

## Update Expense Offline

* Update local SQLite
* Set sync_status = pending
* Add update action to sync_queue

## Delete Expense Offline

* Soft delete locally
* Set sync_status = pending
* Add delete action to sync_queue

## Conflict Handling

If same record is updated locally and remotely:

* Prefer latest updated_at timestamp
* Keep conflict copy if needed
* Show conflict notification if important

# Important Development Rule

Do not make the app depend fully on the backend.

The app must work offline first.

Every major action should happen locally first, then sync with the backend later.

This is the most important rule of the project.
